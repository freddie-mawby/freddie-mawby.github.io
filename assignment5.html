Interactive Choropleth Map

Made using Esri Online Web App Builder

The aim of this assignment was to make an interactive version of our choropleth map from assignment 3. Esri Online is an easy way to share, publish and access datasets that can then be used to enhance the datasets you’re working with. It’s a form of online storage that is very easy to use, but Esri is a service that is very expensive financially. A lot of the data on Esri is publicly available, which means that there are open source alternatives. These alternatives are free of cost (you do not have to pay to use them), but also free of use (you have freedom to edit them for your own needs). With open data, it’s important for it to be FAIR. FAIR stands for Findable, Accessible, Interoperable, and Reusable.

This assignment was also the point in the course where I chose to go back and make changes to assignment 3, my original choropleth map. There were a lot of things that I liked about my original choropleth map: I thought the classification was pertinent, the choice of the Mollweide projection felt as neutral as possible, and centering the map around Europe (the most classic choice of central meridians) also felt like the best way to give an overview of the global state of land degradation. I am well-aware that these choices are still biassed, particularly the choice of central meridians, but these still felt like the most relevant choices within the context of my map, and as such, I kept them in my interactive map as well. As mentioned previously, however, I did want to change my colour scale, and more importantly, not show the countries with no data as having less than 5% land degradation. I implemented these changes in my interactive map.

For my interactive web map, I took the shapefile I obtained from the original choropleth map of SDG indicator 15.3.1, and created an online map using Esri Online Web App Builder. Because this one was interactive, it was important to me that the pop-ups (the information that shows up when you click on a feature) showed the right information. There was some selection involved, since there were many other categories in my attribute table that were not relevant to the map I was making. Now, when you click on a country, you can see its name and the percentage of total land that is damaged. I originally made this map as an Instant App, as opposed to a Web App, which had an interactive legend, allowing the user to click on the categories I had defined and see all the countries that fit into it. I enjoyed this feature, so when I remade this map in the Web App Builder, I chose to include the feature “filter by”, which essentially has the same effect, but allows for more specific filtering (and better visualisation on this website).
